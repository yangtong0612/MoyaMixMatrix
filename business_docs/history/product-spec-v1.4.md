# 彤见矩阵产品说明书

版本：终端操盘版 v1.4  
产品定位：面向商家的 AI 短视频增长操盘系统  
核心口号：让每个卖点都被看见

---

## 第 1 页：产品定位与终端客户

### 1. 产品一句话

彤见矩阵是一套给终端商家使用的 AI 短视频增长操盘系统，帮助商家把商品、门店、套餐、活动和真实卖点，转化为可发布、可测试、可复盘的短视频内容。

它不是单纯的视频生成工具，也不是给代理看的招商系统。产品层面只聚焦一件事：

让商家知道“今天该发什么、怎么表达利益点、发完怎么看结果、下一轮怎么优化”。

### 2. 第一目标客户

第一阶段聚焦：

3-30 家门店的区域餐饮团购商家。

典型客户：

- 火锅、烤肉、自助餐、地方菜、小吃快餐
- 有团购套餐、有门店活动、有持续获客需求
- 已经在抖音、小红书、视频号、美团/抖音生活服务发内容
- 但没有稳定内容团队，也不会系统复盘

后续可扩展到茶饮、烘焙、美业、电商和课程，但产品 MVP 先围绕餐饮团购跑通。

### 3. 商家的真实痛点

商家不是没有剪辑软件，而是每天面临这些真实问题：

1. 不知道该发什么  
   店里有菜品、有套餐、有活动，但不知道怎么变成视频选题。

2. 不会表达利益点  
   套餐便宜、菜品好、环境好，但视频里说不清“用户为什么现在要来”。

3. 内容产能不稳定  
   老板、店员、运营都忙，发视频靠临时想，难以持续。

4. 人工剪辑和代运营太重  
   找剪辑慢，找代运营贵，找达人不稳定。

5. 活动信息容易出错  
   门店、价格、有效期、核销规则一旦说错，容易引发投诉。

6. 发完没有复盘  
   不知道哪条视频有效、哪个标题有效、哪个套餐更能带来点击和核销。

### 4. 产品核心价值

彤见矩阵要给商家的不是“多一个工具”，而是一套短视频日常操盘方法：

- 把套餐变成卖点
- 把卖点变成脚本
- 把脚本变成内容测试包
- 把内容变成发布计划
- 把发布结果变成下一轮优化建议

## 第 2 页：OPC 实际操盘流程

### 1. OPC 在产品里的含义

在产品层面，OPC 不写成商业模式，而落到商家的实际操盘动作：

Offer：今天主推什么套餐/活动。  
Point：这个套餐最打动顾客的利益点是什么。  
Content：把利益点变成可发布、可测试的短视频内容。

商家每一次活动，都可以按这个流程完成一次内容操盘。

### 2. 商家操盘闭环

完整流程：

选择活动/套餐 -> 上传素材 -> AI 提炼利益点 -> 生成内容方向 -> 生成短视频和标题 -> 审核活动信息 -> 发布记录 -> 数据复盘 -> 下一轮优化。

这条流程解决的是商家日常经营中的“不会持续发、不会看结果、不会优化”的问题。

### 3. 输入内容

商家需要输入：

- 门店图
- 菜品图
- 环境图
- 团购套餐名称
- 原价和现价
- 使用规则
- 适用门店
- 有效期
- 顾客评价
- 老板想强调的话
- 历史视频链接或截图

系统把这些内容结构化成：

- 价格利益点
- 场景利益点
- 菜品卖点
- 到店理由
- 稀缺性
- 时间限制
- CTA 行动引导

### 4. 输出内容

系统输出的不是单条视频，而是一次活动的内容测试包：

- 3-5 个卖点方向
- 10-30 条发布版视频
- 30-60 条备用变体
- 标题和封面标题
- 字幕和 CTA
- 发布建议
- 发布台账
- 复盘建议

商家关心的不是“生成多少条”，而是能不能持续测试出更有效的内容。

## 第 3 页：核心功能模块

### 1. 商家首页

商家进入系统后，首先看到：

- 当前主推活动
- 待补充素材
- 待生成内容
- 待发布视频
- 已发布数据
- 风险提醒
- 下一步建议

首页的目标是让商家知道今天该做什么，而不是面对复杂编辑器。

### 2. 套餐活动管理

商家可以创建一个活动卡片：

- 活动名称
- 主推套餐
- 活动价格
- 适用门店
- 有效期
- 核销规则
- 注意事项

系统围绕这个活动生成内容，而不是让商家从空白脚本开始。

### 3. 素材库

素材库按经营场景组织：

- 门店环境
- 菜品图片
- 套餐图片
- 老板/员工口播
- 顾客评价
- 达人素材
- 历史成片
- 品牌元素

系统会提示缺少哪些素材，例如没有门店门头、没有套餐主图、没有顾客评价。

### 4. 利益点提炼

系统自动从套餐和素材中提炼：

- 这条内容适合什么顾客
- 用户为什么现在要来
- 价格有没有吸引力
- 菜品组合有什么亮点
- 场景是聚餐、约会、家庭、夜宵还是午餐
- 是否有时间限制或稀缺感

商家可以确认或修改这些利益点。

### 5. 内容生成

围绕同一个活动，系统生成不同角度：

- 优惠价格版
- 菜品卖相版
- 门店环境版
- 顾客评价版
- 老板推荐版
- 节日活动版
- 距离到店版

每个角度都包含脚本、标题、封面文案、字幕和视频。

### 6. 审核与风险提醒

系统在发布前提醒：

- 价格是否和活动一致
- 有效期是否过期
- 适用门店是否完整
- 是否夸大食品功效
- 是否存在肖像授权问题
- AI 生成内容是否需要标识

这能减少商家因为内容错误产生投诉。

### 7. 发布台账

商家记录每条视频：

- 发布平台
- 发布账号
- 发布时间
- 对应活动
- 对应门店
- 对应套餐
- 视频链接

台账的意义是让后续复盘有依据。

### 8. 数据复盘

早期支持手动录入或截图记录：

- 曝光
- 点赞
- 评论
- 私信
- 团购链接点击
- 核销反馈
- 到店反馈

系统根据结果推荐：

- 哪个卖点继续放大
- 哪个标题可以复用
- 哪个套餐表现更好
- 下一轮该生成什么内容

## 第 4 页：MVP 范围与产品指标

### 1. MVP 必须先做

第一版只做商家最刚需的闭环：

- 活动/套餐创建
- 素材上传
- 卖点提炼
- 内容测试包生成
- 风险提醒
- 发布台账
- 基础复盘

### 2. 暂不做

第一阶段不做：

- 全平台自动发布
- 复杂剪辑特效
- 多行业模板商城
- 自动 CPS 归因
- 大客户私有化
- 完整广告投放系统

### 3. 北极星指标

产品北极星指标：

商家每月完成“生成 -> 发布 -> 复盘”的活动次数。

辅助指标：

- 每个商家创建活动数
- 每个活动生成内容数
- 生成后发布率
- 发布后数据回收率
- 复盘后再次生成率
- 内容返修率
- 商家续费率

### 4. 试点目标

60 天试点目标：

| 指标 | 目标 |
| --- | --- |
| 试点商家 | 20 家餐饮团购商家 |
| 每家活动数 | 1-3 个 |
| 每个活动发布视频 | 10-30 条 |
| 发布数据回收率 | 70% 以上 |
| 次月继续使用 | 50% 以上 |
| 真实案例 | 3-5 个 |

## 第 5 页：产品差异化

### 1. 与剪辑工具不同

剪辑工具解决“怎么剪”，彤见矩阵解决“今天该发什么，发完怎么看结果”。

### 2. 与通用 AI 视频工具不同

通用 AI 工具生成画面，彤见矩阵围绕商家的套餐、价格、门店、活动和核销规则生成经营内容。

### 3. 与代运营不同

代运营靠人交付，商家很难沉淀方法。彤见矩阵把内容操盘变成可复用流程，商家自己的素材、成片和复盘都沉淀在系统里。

### 4. 与数据平台不同

数据平台告诉商家趋势，彤见矩阵直接把趋势和卖点转化为下一轮内容。

### 5. 产品最终愿景

彤见矩阵最终要成为商家的 AI 短视频操盘台：

商家不需要每天从零想选题，也不需要完全依赖外部代运营，而是围绕自己的真实活动和真实卖点，持续生成、发布和复盘内容。
