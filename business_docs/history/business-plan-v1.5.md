# 彤见矩阵商业计划书

版本：分阶段商业叙事版 v1.5  
项目定位：AI 短视频增长培训与授权渠道平台  
核心口号：让每个卖点都被看见

---

## 第 1 页：商业定位与总体路径

### 1. 一句话商业定位

彤见矩阵不是只向终端商家卖软件，而是围绕 AI 短视频增长能力，分阶段构建“OPC 转化、企业培训、终端案例、认证代理、授权渠道、系统消耗”的商业体系。

### 2. 核心商业判断

终端商家需要短视频增长结果，但大多数商家不具备持续学习、执行、发布和复盘的能力。

因此，彤见矩阵的商业路径不是一步到位做大 SaaS，而是分阶段推进：

第一阶段：用 OPC 和终端操盘案例验证需求。  
第二阶段：用企业培训和实战营放大信任。  
第三阶段：用认证代理承接重交付。  
第四阶段：用授权渠道和系统服务费规模化。  
第五阶段：用企业版、白标和数据价值平台化。

### 3. 为什么要分阶段

如果一开始就同时做软件、培训、代理、加盟、白标和数据，会显得不聚焦。

分阶段的好处是：

- 每个阶段有清晰主收入。
- 每个阶段验证下一个阶段的前提。
- 不把远期模式包装成当下收入。
- 投资人能看清楚收入曲线如何演化。

## 第 2 页：阶段一，终端操盘与 OPC 验证

### 1. 阶段目标

0-3 个月，先证明三件事：

1. 商家真的有短视频操盘痛点。
2. 商家愿意为内容测试和操盘结果付费。
3. OPC 能以可控成本获得商家和服务商线索。

### 2. 主要客户

- 区域餐饮团购商家
- 餐饮老板
- 本地生活运营人员
- 想做本地服务的创业者和代运营人员

### 3. 产品交付

面向终端商家交付：

- 7 天内容操盘试点
- 餐饮团购内容测试包
- 活动/套餐卖点梳理
- 10-30 条发布版视频
- 基础发布台账
- 7 天复盘建议

面向 OPC 线索交付：

- 公开课
- 低价训练营
- 商家素材作业
- AI 短视频增长认知课

### 4. 收入结构

| 收入项 | 定价方向 | 目的 |
| --- | --- | --- |
| OPC 公开课 | 9.9-99 元 | 回收获客成本，筛选意向 |
| 7 天实战营 | 399-999 元 | 训练商家整理素材和卖点 |
| 终端试点包 | 999-2999 元 | 验证真实付费 |
| 标杆操盘项目 | 1-3 万元 | 打案例，形成销售素材 |

### 5. 阶段指标

| 指标 | 目标 |
| --- | --- |
| OPC 线索 | 1000-3000 条 |
| 到课率 | 30%-50% |
| 实战营付费 | 100-300 人 |
| 试点商家 | 20 家 |
| 真实案例 | 3-5 个 |
| 月收入 | 10-30 万 |

## 第 3 页：阶段二，企业培训与方法产品化

### 1. 阶段目标

3-6 个月，把第一阶段跑出的操盘方法、案例和系统流程，包装成企业培训和行业实战营。

### 2. 主要客户

- 连锁餐饮品牌
- 商协会
- 园区/产业带
- 本地生活服务商
- 电商企业
- 代运营公司
- 职业培训机构

### 3. 交付内容

企业培训不只是讲课，而是帮组织建立 AI 短视频 SOP：

- AI 短视频增长认知课
- 行业案例拆解
- 商品/套餐卖点训练
- 短视频内容测试 SOP
- 企业内部账号矩阵训练
- 企业专属模板包
- 系统试用
- 后续顾问服务

### 4. 收入结构

| 收入项 | 定价方向 | 目的 |
| --- | --- | --- |
| 企业内训 | 2-20 万/场或项目 | 高客单现金流 |
| 21 天企业实战营 | 1999-5999 元/人 | 批量训练员工 |
| 行业模板包 | 999-9999 元 | 标准化方法输出 |
| 年度顾问 | 5-30 万/年 | 深度服务标杆客户 |

### 5. 阶段指标

| 指标 | 目标 |
| --- | --- |
| 企业培训客户 | 3-10 家 |
| 企业培训复购/转介绍 | 可追踪 |
| 行业案例 | 10-30 个 |
| 可复用课程模块 | 10 个以上 |
| 月收入 | 30-80 万 |

### 6. 阶段价值

企业培训解决两个问题：

- 增强品牌和专业背书。
- 为第三阶段认证代理提供课程、案例和标准。

## 第 4 页：阶段三，认证代理与重交付外移

### 1. 阶段目标

6-9 个月，把终端重交付从总部转移给认证代理。

总部不直接养大量交付人员，而是输出：

- 课程
- 系统
- 模板
- SOP
- 质检
- 品牌
- 结算

### 2. 代理画像

- 本地代运营公司
- 摄影摄像团队
- 团购服务商
- 私域服务商
- 想转型 AI 服务的创业者
- 职业培训机构本地团队

### 3. 代理交付内容

代理向终端商家交付：

- 短视频内容测试包
- 门店素材采集
- 套餐卖点梳理
- 发布协助
- 基础数据复盘
- 月度续费服务

### 4. 平台收入

| 收入项 | 定价方向 | 说明 |
| --- | --- | --- |
| 认证代理营 | 1-5 万/期 | 培训和筛选代理 |
| 代理系统服务费 | 999-9999 元/月 | 工作台、生成、台账 |
| AI 美豆 | 按量计费 | 视频生成、数字人、高清导出 |
| 模板授权 | 999-9999 元/行业包 | 行业打法和模板 |
| 质检督导费 | 按月或按项目 | 控制交付质量 |

### 5. 阶段指标

| 指标 | 目标 |
| --- | --- |
| 认证代理 | 20-50 家 |
| 代理首月签约商家 | 2-5 家/代理 |
| 代理系统活跃率 | 70% 以上 |
| 代理投诉率 | 可控 |
| 代理贡献收入 | 超过总收入 30% |

## 第 5 页：阶段四，授权渠道与系统消耗规模化

### 1. 阶段目标

9-12 个月，把认证代理模型升级为授权渠道体系。

### 2. 渠道类型

| 渠道 | 角色 |
| --- | --- |
| 城市合伙人 | 负责区域招商和督导 |
| 行业合伙人 | 深耕餐饮、美业、电商等行业 |
| 企业渠道 | 商协会、园区、培训机构 |
| 白标服务商 | 用自己的品牌服务客户 |

### 3. 授权产品

| 授权产品 | 定价方向 | 交付内容 |
| --- | --- | --- |
| 城市合伙人 | 10-50 万/年 | 区域授权、招商支持、督导 |
| 行业合伙人 | 10-100 万/年 | 行业模板共建、行业客户共拓 |
| 企业内训授权 | 2-20 万/项目 | 培训、案例、模板、系统试用 |
| 白标系统 | 10-100 万/年 | 品牌化系统、账号体系、私有模板 |

### 4. 平台持续收入

授权不是一次性收费，后续还要持续产生：

- 系统服务费
- AI 生成消耗
- 模板升级费
- 质检督导费
- 项目分润
- 企业培训复购

### 5. 阶段指标

| 指标 | 目标 |
| --- | --- |
| 授权城市/行业合伙人 | 5-10 个 |
| 认证代理 | 150-300 家 |
| 代理贡献收入 | 超过总收入 50% |
| 平台月收入 | 100-300 万 |

## 第 6 页：阶段五，企业版、白标与数据价值

### 1. 阶段目标

12 个月以后，平台从培训授权网络升级为 AI 短视频增长基础设施。

### 2. 远期收入

- 企业版系统
- 白标系统
- API 接入
- 行业数据报告
- 交易归因
- 团购核销分成
- 线索分成

这些模式必须建立在三个基础上：

1. 渠道网络已经有规模。
2. 终端案例和数据持续回流。
3. 系统能承接多角色协同。

### 3. 为什么不能太早做

白标、API、数据分成看起来客单价高，但早期会消耗研发和交付能力。只有当前四个阶段跑通后，这些才是平台价值，而不是定制项目陷阱。

## 第 7 页：产品中台与商业链路的关系

### 1. 产品是商业链路的底座

彤见矩阵的产品中台支撑每个商业阶段：

| 产品模块 | 支撑收入 |
| --- | --- |
| 课程后台 | OPC、训练营、企业培训 |
| 商家操盘台 | 终端试点、案例沉淀 |
| 代理工作台 | 认证代理、授权渠道 |
| 素材库 | 系统服务费 |
| AI 生成 | 美豆和生成消耗 |
| 模板库 | 模板授权 |
| 质检系统 | 督导费和风控 |
| 发布台账 | 复盘服务和数据资产 |
| 结算系统 | 项目分润和渠道管理 |

### 2. 终端商家的角色

终端商家不是唯一收入来源，而是：

- 产品验证场
- 案例来源
- 数据来源
- AI 消耗场景
- 代理招商素材
- 企业培训素材

### 3. 平台总部的长期价值

总部真正沉淀的是：

- 课程体系
- 代理体系
- 行业模板
- AI 生成系统
- 交付 SOP
- 质检规则
- 内容表现数据
- 渠道结算网络

### 3. 核心验证指标

| 指标 | 目标 |
| --- | --- |
| OPC 线索成本 | 可控并可复投 |
| OPC 到课率 | 30%-50% |
| 训练营转化率 | 持续优化 |
| 企业培训成交 | 3-10 家 |
| 认证代理 | 20-50 家 |

| 系统服务费续费 | 可追踪 |
| AI 生成消耗 | 持续增长 |

### 4. 对投资人的核心表达

彤见矩阵不是靠单一产品订阅增长，而是分阶段构建 AI 短视频增长能力的商业网络。

第一阶段用 OPC 和终端案例验证需求；第二阶段用企业培训放大信任；第三阶段用认证代理承接重交付；第四阶段用授权渠道和系统消耗规模化；第五阶段再做企业版、白标和数据价值。
